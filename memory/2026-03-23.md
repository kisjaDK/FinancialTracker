## 2026-03-23

- `app/api/tracker-seats/route.ts` and `app/api/tracker-seats/[seatId]/route.ts`: seat create/update/delete must call `revalidatePath` for `/people-roster` and the dependent finance pages after the mutation. `router.refresh()` alone did not reliably refresh the people roster row after editing a seat, because the server-rendered page data stayed stale until explicit path invalidation.
- `components/layout/theme-toggle.tsx`: the theme dropdown should not participate in SSR hydration. Gate the Radix dropdown behind a `mounted` flag and render a stable placeholder button first; otherwise the initial server/client tree can diverge and shift Radix-generated IDs, which shows up as hydration mismatches on unrelated controls like page `Collapsible` components.
- `components/finance/people-roster-browser.tsx`: the roster list `Planning` date range should render `effectiveStartDate` / `effectiveEndDate` with fallback to imported `startDate` / `endDate`. Using the raw roster dates makes the table disagree with the edit seat dialog after tracker overrides are saved.
- Validation for this change: `npm run build`.
