## 2026-03-20

- `components/finance/seat-editor-dialog.tsx`: hierarchy is now the primary selector in the seat dialog; the hierarchy picker uses richer two-line items and auto-fills `projectCode` and inferred `team`, so those fields no longer need separate visible inputs.
- `components/finance/people-roster-browser.tsx`: default team inference for the seat dialog comes from the most common current team on roster rows matching the same domain, sub-domain, and project code as the selected budget area.

- `lib/finance/queries.ts` and `components/finance/admin-browser.tsx`: role and band reference values are backfilled from current seat/roster data, persisted alongside other maintained seat dropdown values, and exposed as extra maintenance cards on Admin Home.

- `prisma/schema.prisma`, `lib/finance/queries.ts`, and `components/finance/admin-browser.tsx`: `RESOURCE_TYPE` is now a maintained seat reference value, backfilled from current seat/roster data and managed from its own Admin Home card.
- `components/finance/seat-editor-dialog.tsx`: vendor, location, and manager use searchable typeahead pickers that keep the selected value visible in the trigger; resource type now uses maintained dropdown values instead of free text.
- `components/finance/people-roster-browser.tsx`: the main roster table card now uses the `Roster List` label and `#roster-list` anchor instead of the old imported-roster naming.

- `components/finance/seat-reference-values-card.tsx`: roster-select maintenance cards on Admin Home are now collapsible and default collapsed so users can expand only the select list they want to work on.

- `components/finance/admin-browser.tsx`: the hierarchy mapping card now spans the full admin content width, and the Teams table column renders one team per line for readability.
- `components/finance/admin-browser.tsx`: the six hierarchy-mapping input fields now render in a 3-column grid on medium+ screens instead of stretching full width.
