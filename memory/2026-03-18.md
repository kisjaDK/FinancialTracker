## 2026-03-18

- `prisma/schema.prisma`: machine-to-machine auth now uses dedicated `ServiceUser` and `ServiceApiKey` tables instead of overloading `AppUser`; keep API keys one-way by storing only `keyId` plus a SHA-256 hash of the full bearer token.
- `lib/service-users.ts`: service keys use the format `pnd_srv_<12 hex keyId>_<base64url secret>`; lookup is by `keyId`, verification is constant-time against the stored hash, and plaintext keys are only returned from create/rotate responses.
- `app/api/imports/roster/route.ts`: roster import now supports both multipart CSV and JSON payloads on the same route; if an `Authorization: Bearer` header is present it can authenticate a service user, otherwise the route falls back to the normal signed-in viewer flow.
- `lib/roster-api-auth.ts`: API-key auth is intentionally route-local for roster import v1; do not broaden machine credentials to the rest of `requireApiAccess` without an explicit product decision.
- `components/finance/user-admin-browser.tsx`: service-user management lives alongside User Admin and shows generated keys exactly once; UI should never try to read keys back from storage after creation or rotation.
- `lib/finance/imports.ts`: CSV and JSON roster ingestion should share the same persistence path so `RosterImport`, `RosterPerson`, hierarchy mapping errors, and `deriveTrackerSeatsForYear` stay aligned across import sources.
- `app/analysis/page.tsx` and `components/finance/navigation.ts`: the Analysis page is now reserved for `SUPER_ADMIN` users; keep both the page guard and nav minimum role aligned so the route is hidden and direct access is rejected.
- `lib/ai/config.ts` and `lib/ai/tasks/run-analysis-test.ts`: AI config should explicitly report `isConfigured` and missing env vars instead of silently falling back to hard-coded Ollama defaults; the Analysis page should render a non-fatal "not configured" state when `OLLAMA_BASE_URL` or `OLLAMA_MODEL` is absent.
- Validation command set for this change: `npm run db:generate`, `npm test`, and `npm run build`.
