## 2026-03-17

- `app/analysis/page.tsx`: the new Analysis page is dynamic server-rendered and runs a minimal Ollama connectivity check on request, which avoids requiring a live Ollama instance during `next build`.
- `lib/ai/config.ts`: Ollama integration uses env-driven config with `OLLAMA_BASE_URL`, `OLLAMA_MODEL`, `ANALYSIS_TEST_PROMPT`, and `OLLAMA_TIMEOUT_MS`; keep provider access server-side only.
- `lib/ai/providers/ollama.ts`: the repo now uses Vercel AI SDK `generateText` with `ollama-ai-provider-v2` and `createOllama({ baseURL, compatibility: "strict" })` for remote Ollama connections.
- `components/finance/navigation.ts`: `/analysis` is available to `GUEST` users as a finance-shell page for simple AI connectivity verification before adding deeper workflows.
- `scripts/docker-dev-start.sh`: Docker dev startup should detect dependency changes by hashing both `package.json` and `package-lock.json`; relying on timestamps or the persisted `node_modules` volume alone can miss newly added packages.
- `scripts/docker-dev-start.sh`: when Docker detects dependency changes in the mounted workspace, use `npm ci` rather than `npm install` so the container matches `package-lock.json` exactly.
- `scripts/docker-dev-start.sh`: when `.next` is mounted as a Docker volume, never `rm -rf .next`; clear its contents with `find .next -mindepth 1 -maxdepth 1 -exec rm -rf {} +` or the app container can crash with `Device or resource busy`.
